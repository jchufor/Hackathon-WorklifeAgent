# 퇴뽓

**잡무를 줄이고 생각할 시간을 되돌려주는 AI 에이전트.**

**[→ 데모 열기](https://jchufor.github.io/Hackathon-WorklifeAgent/)**

---

## 소개

퇴뽓은 사내 여러 시스템 앞에 자리 잡고, 임직원이 매일 반복하는 행정 잡무를 대신 처리하는 사내 업무 보조 어시스턴트입니다 — 회의실 예약, SVPN 신청, 사원증 재발급, IT 헬프데스크 요청, 셔틀 조회 등.

팀 기획 회의에서 도출한 핵심 인사이트:

> *"문제는 임직원이 일을 모르는 게 아니다. 매번 내부 시스템을 다시 찾고, 다시 확인하고, 다시 제출해야 하는 과정에서 시간이 새는 것이다."*

에이전트는 질문에 답하는 것에서 그치지 않습니다. 올바른 화면을 열고, 입력 항목을 미리 채워 넣고, 최종 확인 클릭 하나만 남겨 놓습니다. 실행과 함께, 참여형 위키가 조직 지식을 지속적으로 축적합니다 — 모든 질문이 잠재적 위키 문서가 되고, 모든 실패가 자동 수정 제안으로 이어집니다.

---

## 핵심 가치 흐름

```
잡무 실행 → 지식 축적 → 매뉴얼 최신화 루프 → 생각할 시간 회복
```

목표는 "8시간짜리 잡무를 1시간으로 줄이는 것"이 아닙니다. 행정 마찰로 잃어버린 시간을 본업·학습·혁신으로 되돌리는 것입니다.

---

## 주요 기능 (프로토타입 범위)

### 1. 오케스트레이터 + 멀티 에이전트 채팅
단일 채팅 인터페이스가 뒤편에서 요청을 적절한 전문 에이전트로 라우팅합니다. 사용자는 하나의 어시스턴트와 대화하지만, 내부적으로는 12개의 전문 에이전트가 협업합니다.

| 에이전트 | 담당 업무 |
|---|---|
| 셔틀 | 시간표 조회, 실시간 지연 정보 |
| 회의실 | 객실 검색, 수용 인원, 화상회의 예약 |
| 복지 | 복지포인트 잔액, 사용 내역 |
| IT 헬프데스크 | SVPN 신청, PC 반출, 웹사이트 예외 처리 |
| 외부교육 | 신청 절차, 대리 승인자 처리 |
| 사원증 | 분실 신고 + 재발급 한 번에 처리 |
| 인사 / 규정 | 승인 체계, 위임 규정 |
| 지식 (위키) | 위키 검색 및 문서 제시 |
| + 4개 추가 | 식당, 비용 정산, 일반, 오케스트레이터 |

**오케스트레이터 인스펙터** (UI에서 확인 가능)는 내부 라우팅 경로를 보여줍니다 — 검토된 에이전트, 최종 선택된 에이전트, 신뢰도, 응답 시간, 사용된 도구. 데모의 투명성 레이어입니다.

### 2. 실행 카드 (검색이 아닌 실행)
지원 업무의 경우, 에이전트는 단순 답변 대신 단계별 실행 카드를 제시합니다. 각 단계는 카드에서 바로 실행 가능합니다 (양식 자동 입력 → 제출 → 알림). 기존 백엔드 시스템은 그대로 두고, 에이전트는 그 위에 간소화된 프런트엔드 레이어를 추가합니다.

### 3. 참여형 위키
위키피디아 방식의 사내 지식 베이스로, 누구나 절차 문서를 작성·수정·보완할 수 있습니다. 모든 콘텐츠는 명시적으로 비공식으로 표시되어, 단일 관리자에게 정확성 유지 부담이 집중되지 않습니다.

- **검토 대기열**은 반복 질문이나 기록된 실패를 바탕으로 AI가 제안한 위키 업데이트를 표시합니다
- 위키 패널에서 바로 제안을 승인하거나 거부할 수 있습니다
- 여러 번 들어오는 질문은 위키 페이지 후보가 됩니다

### 4. 커뮤니티 ASK 보드
임직원이 질문을 올리면, 관련 에이전트 여러 개가 동시에 다양한 관점으로 답합니다. 좋은 답변이 달린 게시물은 클릭 한 번으로 위키 문서로 전환할 수 있습니다 ("위키화").

### 5. 내 업무 패널
에이전트가 처리한 모든 작업의 기록 — 절약 시간 추정치, 상태 추적, 활성 에이전트 수를 한눈에 확인할 수 있습니다.

## 아키텍처

### 멀티 에이전트 설계 (프로덕션 계획)

```
사용자 요청
    ↓
오케스트레이터 에이전트  ←→  의도 분류, 라우팅
    ↓
전문 서브 에이전트 (예: IT, 셔틀, 회의실)
    ↓
위키 검색 서브 에이전트  ←→  관련 절차 문서
    ↓
응답 + 실행 카드
    ↓
지식 업데이트 서브 에이전트  ←→  실패 기록, 수정 제안 생성
```

각 서브 에이전트는 자체 `agent.md` 명세 파일로 정의됩니다. 오케스트레이터는 이 파일을 읽어 라우팅을 결정합니다. 이 설계는 FabriX/사내 API 배포를 목표로 하지만, 해커톤 프로토타입은 데모 안정성을 위해 전체 의도 매칭 및 라우팅 로직을 클라이언트 사이드에서 실행합니다.

> **참고:** FabriX 내부 LLM 통합은 불안정한 것으로 확인되었습니다 (외부 API로는 작동하지만, 사내망 내부에서는 신뢰하기 어려움). 프로토타입은 네트워크 의존성 없이 오프라인에서도 작동하도록 라우팅 로직을 자체 포함 방식으로 유지합니다.

### 프로토타입 기술 스택

| 레이어 | 선택 | 이유 |
|---|---|---|
| UI | 단일 HTML 파일 (`WorkLife Agent.html`) | 더블클릭으로 실행, 서버·네트워크 불필요 |
| 프레임워크 | React 18 + Babel standalone (로컬) | 로컬 복사본 — `react.min.js`, `react-dom.min.js`, `babel.min.js` |
| 스타일 | `worklife.css` (동일 폴더, 외부 파일) | 시스템 폰트만 사용 — CDN 없음, 제한 네트워크 환경에서도 작동 |
| 다국어 | 한국어/영어 자동 감지 + 토글 | `localStorage` 저장, `wl-lang-change` 이벤트 |
| 데이터 | 인라인 JS 시드 데이터 | 시나리오, 에이전트, 게시글, 위키 페이지, 업무 이력 |
| LLM (프로덕션 목표) | Gemini API (현재) / 로컬 Ollama | 팀 Gemini API 키 사용 중; 망분리 환경용 Ollama + 양자화 모델 검토 |

## 프로토타입 실행 방법

1. `WorkLife Agent.html`, `worklife.css`, `react.min.js`, `react-dom.min.js`, `babel.min.js`를 **같은 폴더**에 유지합니다.
2. `WorkLife Agent.html`을 더블클릭 — 서버 없이 브라우저에서 바로 열립니다.
3. 인터넷 연결 불필요. 모든 에셋은 로컬입니다.

**데모 시 추천 순서:**
- 요청을 직접 입력하거나 예시 프롬프트 클릭 ("SVPN", "회의실", "복지포인트", "셔틀", "사원증" 등)
- 라우팅 펄스 애니메이션으로 에이전트 선택 과정 확인
- "처리 담당 · [에이전트]" 태그 클릭 → 오케스트레이터 인스펙터 패널 열기
- 커뮤니티, 위키, 내 업무 탭 전환
- 우측 상단 언어 버튼으로 한국어 / 영어 토글
- 페르소나 버튼 전환 (신입 / 경력 / 팀장) → 각기 다른 예시 프롬프트 확인

## 문제 정의

> *임직원은 자신의 업무를 어떻게 해야 하는지 알고 있다. 하지만 올바른 시스템을 찾고, 현재 절차 경로를 파악하고, 파편화된 불편한 사내 도구를 통해 요청을 제출하는 과정에서 지속적으로 시간을 잃는다.*

**주요 불편 사례:**
- SVPN, 근태 앱, 사원증 등 기본 신청 절차가 분산되어 있고 중앙화된 안내 없음
- IT 헬프데스크 요청(PC 반출·웹사이트 예외 등)이 담당자별로 분리되어 창구가 불명확
- 회의실 예약 시 타 사이트·화상회의 연동 등 여러 단계 클릭 필요
- 외부교육 승인 체계가 불투명하고 대리 승인 절차 안내 없음
- 공식 매뉴얼은 업데이트가 드물어, 시스템 변경 시 경력직도 동일한 마찰을 겪음

신입만의 문제가 아닙니다. 시스템이 바뀌거나 자주 하지 않는 업무를 시도할 때마다 모든 임직원이 같은 마찰을 반복합니다.

## 데이터 전략

| 출처 | 접근 방식 |
|---|---|
| 참여형 위키 | 주요 — 크라우드소싱, 임직원 편집 가능, 명시적 비공식 표시 |
| 질문 로그 | 에이전트 자동 수집, 익명화, 마찰이 높은 주제 파악 |
| 실패 로그 | 잘못된 절차 경로 신고 시 자동 생성 |
| 공식 HR/조직 시스템 | **의도적으로 최소화** — 권한·기밀 리스크; 프로토타입에 미통합 |

위키 콘텐츠의 비공식 면책 표시는 한계가 아닌 의도적 설계 선택입니다. 단일 시스템 관리자에게 정확성 유지 부담이 집중되는 구조를 없애고, 지식 베이스가 자유롭게 성장할 수 있도록 합니다.
